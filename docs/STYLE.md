# Стиль документации

Именование файлов

- Латиница, нижний регистр, слова через `_`.
- Префиксы по тематике: `spec_` (спецификация), `gdd_` (общий дизайн), `pve_` (PvE/системы), `guides_` (гайды), `moscow_` (приоритезация) и т. п.
- Русский язык в тексте; имена файлов — латиница (допускается транслитерация).

Кодировка и формат

- Кодировка: UTF‑8 (без BOM), окончания строк: LF.
- Заголовок H1 = название документа. Следом: краткое описание, Статус/Версия, Ответственный.
- Разделы упорядочивать: Обзор → Термины → Требования → Алгоритмы/Модель → Данные/Конфиги → Тесты/Верификация → UX/HUD (если применимо) → Changelog.

Версионирование

- Один «живой» документ на тему; внутри — `## Changelog`.
- Разные режимы (например, Coupled/Decoupled) — разные документы с независимыми версиями.

Кросс‑ссылки и источники

- Ссылки — относительные пути внутри `docs/`.
- Для внешних форматов (PDF/DOCX) указывать «Источник: <файл/ссылка>» и краткий конспект.

Шаблон спецификации

```
# <Название>
Статус: <актуален/черновик/архив>
Версия: <x.y.z> (дата)
Ответственный: <имя/ник>
Источник: [<оригинал>](relative/or/link)

## Обзор
Короткое описание цели и ожидаемого результата.

## Термины и обозначения
Список ключевых терминов и сокращений.

## Требования
- Функциональные
- Нефункциональные

## Алгоритмы / Модель
Подразделы, формулы, псевдокод, диаграммы.

## Данные и конфигурация
Схемы, параметры, единицы измерения, диапазоны.

## Тесты и верификация
Метрики/кейсы, критерии приемки, способ измерений.

## UX / HUD (если применимо)
Сигналы/индикаторы, телеметрия, иконки, взаимодействие.

## Changelog
- x.y.z: краткие изменения

---

## Полный текст (архив)
Оригинал для справки (не редактируется, используется для миграции/сверки).
```
