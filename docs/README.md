# Документация U2

Этот каталог хранит рабочую документацию по проекту **Universe Unlimited (U2)**: геймдизайн, технические спецификации, PvE‑концепции, гайды и архивные материалы.

Полный навигационный индекс см. в файле: **[INDEX.md](INDEX.md)**.

## Структура документации

### GDD — Game Design Documents
- Директория: `gdd/`
- Назначение: высокоуровневые геймдизайн‑документы, описывающие видение, целевую аудиторию, игровые петли и прогрессию.
- Пример: `gdd/gdd_overview.md` — общий обзор GDD по U2.

### Specs — технические спецификации
- Директория: `specs/`
- Назначение: формальные спецификации систем и режимов, используемые как единый источник правды для разработки.
- Состав:
  - полётные режимы (Coupled / Decoupled) — `specs/spec_pilot_assist_coupled.md`, `specs/spec_flight_decoupled.md`;
  - геймплейные спеки и формулы боя для ветки v0.8.x — `specs/gameplay/...`;
  - архитектура, техстек, конфиги кораблей, визуальный стиль — `specs/tech/...`;
  - отчёты по аудиту документации — `specs/audit/...`.
- Централизованный каталог спецификаций см. в `specs/README.md`.
- Минимальный план реализации (ветка `gpt-5_1-repo-build`) описан в `specs/gameplay/spec_u2_dev_plan_v086_minimal.md` (этап M0.1 — репозиторий и сборка).

### PvE — режимы PvE
- Директория: `pve/`
- Назначение: аналитика и дизайн PvE‑контента — кампании, типы игроков, обзор референсов.

### Guides — руководства и практики
- Директория: `guides/`
- Назначение: рабочие заметки, гайды по процессам, best practices для разработки и продакшна U2.
- Статус: в активной доработке; структура может меняться.

### Стиль документации
- Файл: `STYLE.md`
- Содержит правила оформления:
  - формат заголовка H1 и шапки документа;
  - требования к кодировке и переносам строк;
  - соглашения по именованию файлов (префиксы `spec_`, `gdd_`, `pve_`, `guides_`, `moscow_` и т.д.);
  - рекомендуемые разделы (UX/HUD, Changelog и др.).

## Конвертированные документы (_converted/)

Часть старых документов хранится в виде PDF/DOCX и была конвертирована в Markdown:

- `moscow.md` — матрица MoSCoW для общих требований.
- `moscow-pve.md` — MoSCoW для PvE‑части.
- `pve.md` — исходный документ с общим описанием PvE‑режимов.
- `mmo.md`, `mmo-2.md` — заметки по MMO‑подходам.
- `document.md`, `document-2.md` — общие заметки по документированию и структуре требований.

Оригиналы таких файлов лежат вне `docs/` и могут отличаться по содержанию.

## Архив

- Директория: `archive/`
- Назначение: устаревшие или зафиксированные срезы документации (старые версии архитектуры, режимов, Definition of Fun и т.п.).
- Входные точки:
  - `ARCHIVE.md` — обзор содержимого архива;
  - `archive/README.md` — детальный каталог архивных документов.

## Общие правила для файлов

- Кодировка: **UTF‑8 без BOM**, переносы строк: **LF**.
- Все новые документы должны иметь осмысленный заголовок H1 и краткую шапку (тип документа, версия, краткое описание, ссылки).
- Изменения, влияющие на продукт, по возможности фиксируются в разделах `Changelog` соответствующих спецификаций.
